个人项目实战：二维码签到系统 前端开发总结



一、个人职责说明
在本次二维码签到系统团队开发中，我负责前端开发工作，包括页面布局设计、样式美化、动态交互逻辑编写、后端接口对接、页面防缓存处理、功能调试与用户体验优化。独立完成整套可视化签到页面，实现二维码动态加载、手动刷新、适配浏览器展示，为项目提供完整的用户交互界面。



二、前端技术栈
本项目采用轻量化原生前端技术，无需框架，降低部署成本，适配课程设计小型项目：
- 结构搭建：HTML5 搭建页面主体结构
- 样式美化：原生 CSS3 实现居中布局、按钮悬浮效果、卡片样式
- 交互逻辑：原生 JavaScript 实现动态刷新、防缓存、页面加载事件
- 接口对接：图片请求方式对接后端二维码生成接口



三、前端页面整体设计思路
为贴合校园签到系统简洁、正式、易用的风格，采用了居中卡片式布局，页面无冗余元素，核心功能突出：二维码展示区、功能提示区、刷新按钮
同时针对开发场景优化：添加浏览器防缓存机制、页面自动加载逻辑，解决后端二维码刷新不更新、首次空白等问题。

    function refreshQr(){
        let time = new Date().getTime();

    }
</script>

五、前端核心功能与难点解决
5.1 页面自动渲染功能
通过window.onload 页面加载事件，实现进入页面自动加载二维码，无需手动刷新，提升使用体验。
5.2 二维码防缓存问题解决
浏览器会缓存静态图片，导致后端更新二维码后前端不刷新。我通过拼接时间戳参数，让每次请求地址唯一，强制浏览器重新请求最新二维码，彻底解决缓存bug。
5.3 前后端联调对接
前端通过img标签直接请求后端 /sign/create 接口，接收后端返回的PNG图片流，实现动态渲染；点击刷新按钮可随时获取最新时效二维码，完美对接后端5分钟过期机制。
5.4 用户体验优化
添加按钮悬浮变色、卡片阴影圆角、功能提示文字，页面整洁美观，操作直观，适配电脑端展示，满足课程设计项目展示需求。



六、前端配合后端的业务逻辑
- 页面访问：访问 /sign/index 路由，后端跳转当前前端页面；
- 二维码生成：前端请求后端接口，动态渲染时效二维码；
- 签到交互：手机扫码自动携带Token与活动参数请求后端签到接口；
- 时效配合：前端刷新按钮配合后端动态令牌，实现防代签效果。
